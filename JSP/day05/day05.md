
### 1. GET/POST 전송방식

    1-1. GET 방식
    - 서블릿에 데이터를 전송할 때 데이터가 URL뒤에 name=value 형태로 전송
    - 보안에 취약
    - 전송 가능 데이터 최대 255자
    - 기본 전송 방식으로 사용이 쉬움
    - doGet()로 처리

    1-2. POST 방식
    - 서블릿에 데이터를 전송할 때 TCP/IP 프로토콜 데이터의 HEAD영역에
    숨겨진 채 전송
    - 보안에 유리
    - 전송 데이터 용량 무제한
    - 전송 시 서블릿에서 또 다시 가져오는 작업이 필요하므로 GET방식보다
    처리 속도가 느림
    - doPost()로 처리
